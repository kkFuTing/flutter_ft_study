## 环境配置

FVM 是什么
**FVM（Flutter Version Management）**就是“Flutter 版本管理器”，作用类似 Node 的 nvm、Java 的 jenv：

项目级固定 Flutter 版本：每个项目用自己指定的 Flutter（比如你这个项目固定 3.35.4），不会影响你电脑上其它项目继续用 3.38.3。
团队一致：大家 fvm use 3.35.4 后，用同一套 Flutter SDK 编译，能最大程度避免你说的“同命令但 UI/行为不一样”。
它怎么工作（你需要理解的只有一条）
安装多个 Flutter SDK 到本机缓存
在项目目录里写一个“这个项目用哪个版本”的配置
之后在项目里用 fvm flutter ... 来运行对应版本的 Flutter
如果你想立刻把当前项目对齐到同事的 3.35.4，我可以按你现在的 Windows 环境给你一套最短命令（不会改你全局 Flutter）。

## 安装flutter sdk版本

fvm install 3.35.4

fvm use 3.35.4 --force
fvm flutter --version
fvm flutter clean
fvm flutter pub get
fvm flutter build apk --release
