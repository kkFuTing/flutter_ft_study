# Flutter 学习指南

## 📚 学习路径（按顺序学习）

### 第一阶段：基础概念（第1-2周）

#### 1. 理解Flutter核心概念
- **Widget是什么**：Flutter中一切都是Widget
  - StatelessWidget：无状态组件
  - StatefulWidget：有状态组件
- **MaterialApp 和 Scaffold**：应用的骨架
- **热重载（Hot Reload）**：开发时的神器

#### 2. 基础Widget学习
从 `lib/examples/01_basic_widgets.dart` 开始学习：
- Text：文本显示
- Container：容器
- Row/Column：横向/纵向布局
- Image：图片显示
- Icon：图标
- Button：按钮（ElevatedButton, TextButton, OutlinedButton）

### 第二阶段：布局和样式（第3-4周）

#### 3. 布局Widget
学习 `lib/examples/02_layouts.dart`：
- **Row & Column**：线性布局
- **Stack**：层叠布局
- **Expanded & Flexible**：弹性布局
- **Padding & Margin**：内边距和外边距
- **Center & Align**：对齐方式
- **SizedBox**：尺寸控制
- **ListView**：列表滚动
- **GridView**：网格布局

#### 4. 样式和主题
- Colors：颜色使用
- TextStyle：文本样式
- ThemeData：主题配置
- 使用装饰（Decoration）美化组件

### 第三阶段：交互（第5-6周）

#### 5. 用户交互
学习 `lib/examples/03_interactions.dart`：
- **StatefulWidget 和 setState**：状态管理基础
- **GestureDetector**：手势检测
- **表单输入**：TextField, TextFormField
- **点击事件**：onPressed, onTap
- **页面导航**：Navigator.push/pop

### 第四阶段：进阶（第7-8周及以后）

#### 6. 路由和导航
学习 `lib/examples/04_navigation.dart`：
- **Navigator.push/pop**：基本页面跳转和返回
- **带返回值的跳转**：使用 await 获取页面返回的数据
- **命名路由**：在 MaterialApp 中配置 routes
- **底部导航栏**：BottomNavigationBar 的使用
- **抽屉导航**：Drawer 侧边栏的使用

**关键知识点**：
- `Navigator.push`：跳转到新页面
- `Navigator.pop`：返回上一页
- `Navigator.pushNamed`：使用路由名称跳转
- 页面间传递参数：通过构造函数或 arguments

#### 7. 状态管理（Provider）
学习 `lib/examples/05_state_management.dart`：
- **Provider 基础**：什么是 Provider，为什么要用
- **ChangeNotifier**：创建状态模型
- **Consumer**：监听状态变化并自动更新UI
- **MultiProvider**：提供多个状态管理服务
- **Provider.of**：另一种获取状态的方式

**关键知识点**：
- `ChangeNotifier`：需要继承的基类
- `notifyListeners()`：通知监听者数据变化
- `Consumer<Model>`：自动重建的Widget
- `Provider.of<Model>(context)`：直接获取状态

**其他状态管理方案**（进阶）：
- GetX：更简洁的语法
- Riverpod：Provider的改进版本
- Bloc：事件驱动的状态管理

#### 8. 网络请求
学习 `lib/examples/06_network_request.dart`：
- **HTTP 请求**：使用 http 包发送请求
- **GET 请求**：获取数据
- **POST 请求**：提交数据
- **JSON 解析**：使用 `json.decode()` 解析响应
- **错误处理**：try-catch 处理网络异常
- **加载状态**：显示 Loading 指示器

**关键知识点**：
- `http.get()`：发送 GET 请求
- `http.post()`：发送 POST 请求
- `json.encode()`：将对象转为JSON字符串
- `json.decode()`：将JSON字符串转为对象
- `Uri.parse()` 或 `Uri.https()`：构造URL

**常用API测试网站**：
- JSONPlaceholder：免费的测试API
- httpbin.org：HTTP测试服务

#### 9. 数据持久化
学习 `lib/examples/07_data_persistence.dart`：
- **SharedPreferences**：简单的键值对存储
- **保存数据**：setString、setInt、setBool 等
- **读取数据**：getString、getInt、getBool 等
- **清除数据**：clear() 方法
- **应用生命周期**：initState 中加载数据

**关键知识点**：
- `SharedPreferences.getInstance()`：获取实例
- `setString(key, value)`：保存字符串
- `getString(key)`：读取字符串（可能为null）
- `??` 运算符：提供默认值
- 适合存储：用户设置、简单配置、小量数据

**其他数据持久化方案**（进阶）：
- **sqflite**：SQLite数据库，适合复杂数据结构
- **Hive**：NoSQL数据库，性能好
- **文件存储**：适合大量数据或文件

## 🎯 实践建议

### 每日练习
1. **修改现有代码**：尝试修改 `main.dart` 中的样式、颜色、文字
2. **运行示例代码**：学习 `lib/examples/` 目录下的示例
3. **尝试热重载**：
   - 修改代码后按 `Ctrl+S`（或 `Cmd+S`）
   - 观察变化立即出现在模拟器/设备上

### 第一个小项目建议
完成基础学习后，可以尝试做一个：
1. **待办事项（Todo）应用**
   - 添加任务
   - 删除任务
   - 标记完成
   - 列表显示

2. **天气查询应用**
   - 输入城市名
   - 调用天气API
   - 显示天气信息

## 💡 学习技巧

1. **边学边做**：看完一个知识点，立即写代码实践
2. **修改示例**：不要只复制粘贴，尝试修改参数看效果
3. **理解原理**：理解为什么这样写，而不是死记硬背
4. **遇到问题**：善用搜索引擎和官方文档
5. **做笔记**：记录常用Widget的用法和参数

## 🚀 快速开始

1. 运行项目：
   ```bash
   flutter run
   ```

2. 查看示例：
   - 修改 `main.dart` 中的 `home` 属性
   - 例如：`home: BasicWidgetsExample()`
   - 保存后热重载即可看到效果

3. 学习顺序：
   ```
   示例1：01_basic_widgets.dart（基础组件）
   → 示例2：02_layouts.dart（布局）
   → 示例3：03_interactions.dart（交互）
   → 示例4：04_navigation.dart（路由和导航）
   → 示例5：05_state_management.dart（状态管理）
   → 示例6：06_network_request.dart（网络请求）
   → 示例7：07_data_persistence.dart（数据持久化）
   ```

## ⚠️ 常见问题

### Q: 修改代码后没有看到变化？
A: 尝试热重载（Hot Reload），如果还不行，尝试热重启（Hot Restart）

### Q: 不知道某个Widget怎么用？
A: 查看官方文档，或按 `Ctrl+点击` Widget名称查看源码注释

### Q: 报错怎么办？
A: 
1. 仔细看错误信息
2. 检查括号是否匹配
3. 检查导入的包是否正确
4. 运行 `flutter pub get` 安装依赖
